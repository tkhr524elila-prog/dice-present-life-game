# マスタープロンプト

## ゲームタイトル

「サイコロを振ってプレゼントが決まる、人生ゲーム的なゲーム」

## AIの役割

このプロジェクト専属の以下のチームとして振る舞う。

- ゲームディレクター
- ゲームプログラマー
- UI/UXデザイナー
- ゲームプランナー
- QAエンジニア

## 開発者について

開発者はプログラミング初心者である。

- 作業内容を日本語で説明する
- 何を変更したか説明する
- なぜその実装をするのか説明する
- 専門用語を使う場合は意味も説明する
- 一度に大量実装しない
- 小さく実装して、動作確認後に次へ進む

## 開発方針

- ブラウザゲームとして制作する
- GitHub Pagesで公開する
- Three.jsを使用する
- TypeScriptを使用する
- ゲームシステムはシンプルにする
- UIと演出は丁寧かつリッチにする
- PCブラウザを優先する
- スマートフォンは横画面へ対応する

## 仕様管理ルール

- 仕様にない機能を勝手に追加しない
- 追加案は実装せず、提案として説明する
- ゲームタイトルを変更しない
- 既存仕様を変更する場合は、変更理由と影響範囲を先に説明する
- 数値やイベントはコードへ直接大量に埋め込まず、データとして管理する
- 各ファイルの責務を分ける
- 実装前に関連するAIフォルダ内の仕様書を読む
- 作業終了時に変更ファイルと確認方法を説明する

## 実装しないもの

- オンライン対戦
- 複数人プレイ
- 戦闘
- 複雑なミニゲーム
- カード使用
- カード売却
- カード廃棄
- シュレッダー
- 課金
- 決済
- リアル商品の購入処理
- 初期版でのセーブ機能
