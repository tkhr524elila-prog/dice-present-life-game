# イベントデータ

## ゲームタイトル

「サイコロを振ってプレゼントが決まる、人生ゲーム的なゲーム」

## Phase 7のイベント管理方針

- ゲーム進行は論理100マス、3Dマップは物理120マスとする。
- 第3章は41-A～60-Aと41-B～60-Bを両方配置し、プレイヤーは選択した片方だけを通る。
- 通常イベントの内容、ステータス効果、カード付与は固定する。
- マップ進行中のイベントでランダム抽選を行うのはプレゼント抽選マスだけとする。
- イベントにカードが設定されている場合は100%付与し、「カードなし」の場合は付与しない。
- 交通事故マスへ停止した場合は必ず事故が発生する。
- ゴール後のNISA抽選はマスイベントではなく契約精算として扱う。
- Phase 6までの未使用イベントは削除せず、Phase 7用の差し替え候補・比較・テスト用の予備イベントとして保持する。
- 100進行度とA・B両ルートのポイント・健康・愛情は本書で確定し、P7-11ではシミュレーション結果に基づく調整だけを行う。
- カードを確定付与するイベントは、原則としてカード相当額を同時にポイントから引かない。

## プレゼント抽選マス

- 論理配置数：20か所
- 物理配置数：22個。進行度46と56をA・B両方へ置くため2個増える
- プレイヤー向け名称：「プレゼント抽選」
- 最初に実際に停止した抽選では、通常抽選を行わず、すき家券1枚を確定付与する
- 通過した場合は抽選せず、確定付与済み状態も変更しない
- 2回目以降は愛情0～100の連続抽選率を使用する

## マップ移動処理

- 強制ストップマスを通過する出目の場合、そのマスで停止し、残りの移動数を破棄する。
- 強制ストップイベント処理後の次ターンは、停止したマスから開始する。
- 強制ストップイベントは、1回のプレイ中に1回だけ発生させる。
- 40で恋愛ルートを確定し、41から選択ルートの物理マスへ進む。
- 60-Aまたは60-Bの次は61へ進む。
- 100を超える出目の場合は100で停止し、余った移動数を破棄する。
- 出目をぴったり合わせる必要はなく、100へ到達した時点でゴール精算を開始する。

以下に残す既存イベント表は、文章・数値の比較とP7-02での移植候補として保持する。そこに記載された10%、20%、30%、50%などの確率はPhase 7の有効仕様ではない。Phase 7へ採用するイベントはカード付与率を0%または100%へ変換する。

## イベントデータの項目

各イベントは以下の項目を持つ。

- イベントID
- カテゴリ
- タイトル
- 本文
- ポイント変動
- 健康変動
- 愛情変動
- 付与カード
- カード付与方式（なし／確定）。旧イベント素材では比較用の旧確率を保持する
- 対象ルート
- 職業補正対象か
- 演出メモ

`なし`はカード付与がないこと、`全ルート`は選択ルートに関係なく発生可能であることを表す。

## Phase 7の職業補正ルール

- 外資：仕事プラスポイント×1.5、仕事の健康減少量×1.3、家族の愛情増加量×0.8。
- 代理店：仕事プラスポイント×0.8、仕事の健康減少量×0.7、家族の愛情増加量×1.2。
- マイナスポイント、健康増加、愛情減少には倍率を適用しない。
- 計算結果は四捨五入して整数にする。
- 代理店は「安定した収入を得ながら、健康や家族との時間も守りやすい働き方」、外資は「成果によって大きな収入を狙える働き方」と表現する。

## イベント効果の適用ルール

- ゲーム開始時のポイントは1,000ポイントとする。
- ポイントはイベント適用後に0未満でも保持し、上限も設けない。
- 健康と愛情はイベントで変動するたびに0～100へ補正する。
- 内部データと画面表示のどちらにも、健康・愛情が0未満または100超過の状態を保持しない。
