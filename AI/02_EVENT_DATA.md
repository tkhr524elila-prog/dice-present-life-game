# イベントデータ

## ゲームタイトル

「サイコロを振ってプレゼントが決まる、人生ゲーム的なゲーム」

## Ver1.0のイベント管理方針

- Ver1.0では、全60マスに直接割り当てた固定イベントを使用する。
- 通常イベントマスへイベントをランダム挿入しない。
- ランダム処理は、プレゼントイベントのカード抽選と、明示された確率イベントだけに使用する。
- 60マスへ直接割り当てていないイベントは削除せず、「予備イベント」として保持する。
- 予備イベントは、将来のイベント差し替え候補、文章や数値の比較、Ver2.0以降の追加イベント、テスト用イベントとして使用する。

## マップ移動処理

- 強制ストップマスを通過する出目の場合、そのマスで停止し、残りの移動数を破棄する。
- 強制ストップイベント処理後の次ターンは、停止したマスから開始する。
- 強制ストップイベントは、1回のプレイ中に1回だけ発生させる。
- マス60を超える出目の場合はマス60で停止し、余った移動数を破棄する。
- 出目をぴったり合わせる必要はなく、マス60へ到達した時点でゴール精算を開始する。

### すき家券の確定付与

- マス4固定ではなく、ゲーム中に最初に実際に停止したプレゼントイベントを対象とする。
- 最初の停止時は通常抽選を行わず、すき家券1枚を確定付与する。
- プレゼントイベントマスを通過しただけでは付与しない。
- 内部状態「すき家券確定付与済み」を管理し、初期値を「いいえ」とする。
- 最初の停止後に「はい」へ変更し、その後は愛情値に応じた通常抽選を行う。

## イベントデータの項目

各イベントは以下の項目を持つ。

- イベントID
- カテゴリ
- タイトル
- 本文
- ポイント変動
- 健康変動
- 愛情変動
- 付与カード
- カード付与確率
- 対象ルート
- 職業補正対象か
- 演出メモ

`なし`はカード付与がないこと、`全ルート`は選択ルートに関係なく発生可能であることを表す。

## 職業補正ルール

- 仕事イベントのプラスポイントだけに、外資系保険会社は×1.5、地元の保険代理店は×0.8を適用する。マイナスポイントには適用しない。
- 仕事イベントの健康減少量だけに、外資系保険会社は×1.3、地元の保険代理店は×0.7を適用する。健康増加には適用しない。
- 家族イベントの愛情増加量だけに、外資系保険会社は×0.8、地元の保険代理店は×1.2を適用する。愛情減少には適用しない。
- すべての倍率計算結果は四捨五入して整数にする。

## イベント効果の適用ルール

- ゲーム開始時のポイントは1,000ポイントとする。
- ポイントはイベント適用後に0未満でも保持し、上限も設けない。
- 健康と愛情はイベントで変動するたびに0～100へ補正する。
- 内部データと画面表示のどちらにも、健康・愛情が0未満または100超過の状態を保持しない。
