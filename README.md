# サイコロを振ってプレゼントが決まる、人生ゲーム的なゲーム

サイコロで3Dの人生マップを進み、現実的な人生イベントや選択を体験する1人用ブラウザゲームです。ゴールでは、歩んだ人生に応じた現金額と景品が決まります。

## 公開ページ

[ゲームをブラウザで遊ぶ](https://tkhr524elila-prog.github.io/dice-present-life-game/)

GitHub Pagesの初回設定中や再公開処理中は、一時的に表示できない場合があります。

## 100マス版の特徴

- ゲーム進行は1～100、3Dマップには恋愛A/Bルートを両方含む120物理マスを配置
- 第3章の恋愛選択で、遊び人ルート（41-A～60-A）または純愛ルート（41-B～60-B）へ分岐し、進行度61で合流
- 王様のチュートリアルで、お金・景品カード・愛情・健康の役割を説明
- 愛情0～100に応じてカード抽選率が連続的に変化
- 健康0～100に応じて最終現金倍率が70～130%の間で連続的に変化
- NISA1枠目へ加入済みなら「分散投資！NISA2枠目」を選択可能
- ひろむ、れいな、田中さん、平さん、菊田さん、かな、いとの人物イベント
- プレイ時間の目標は20～30分

## 5章構成

1. 第1章「青春の草原」：進行度1～20
2. 第2章「社会人の城下町」：進行度21～40
3. 第3章「欲望の夜街」：進行度41～60（A/B分岐）
4. 第4章「家族の王国」：進行度61～80
5. 第5章「人生の山脈」：進行度81～100

## 遊び方

1. 「ゲームをはじめる」を押し、王様の説明を読みます。
2. 「サイコロを振る」を押し、出目の数だけマップを進みます。
3. イベントや人生の選択を確認し、「次へ」で進めます。重要な選択には確認画面があります。
4. プレゼント抽選や確定イベントで得たライフカードと、これまでの出来事を記録した人生ノートを確認できます。
5. 進行度100へ到達すると、居候・トラブル・NISA・医療保険・健康倍率の順で精算され、最終現金額と景品一覧が表示されます。

## 必要なもの

- PCブラウザ（優先環境）
- または横画面にしたスマートフォン
- ローカル開発を行う場合はNode.js 22とnpm

## ローカルで起動する方法

ターミナルでリポジトリの`game`フォルダへ移動し、依存関係を準備して開発サーバーを起動します。

```bash
cd game
npm install
npm run dev
```

表示されたURL（通常は `http://localhost:5173/dice-present-life-game/`）をブラウザで開きます。終了するときは、起動中のターミナルで `Control + C` を押します。

## ビルドと公開前確認

ビルドは、開発用コードをブラウザ公開用のファイルへまとめる処理です。

```bash
cd game
npm run build
npm run preview
```

ビルド成功後は`game/dist`へ公開用ファイルが作られます。プレビューの標準URLは `http://127.0.0.1:4173/dice-present-life-game/` です。

## 自動シミュレーション

固定した乱数シードで多数のプレイを再現し、現金・景品・ステータスのバランスを確認できます。

```bash
cd game
npm run simulate -- --seed=20260714
```

標準設定では、職業・恋愛A/B・NISA 0/1/2枠・医療保険・自動車保険を組み合わせた固定48パターンを各1,000回と、ランダム選択10,000回の合計58,000回を実行します。

## GitHub Pagesへの公開

`.github/workflows/deploy.yml`が、`main`ブランチへのPush後に`game`フォルダで依存関係を準備し、ビルド結果の`game/dist`をGitHub Pagesへ公開します。

初回だけ、GitHubのリポジトリ画面で次を確認してください。

1. `Settings`を開きます。
2. 左側の`Pages`を開きます。
3. `Build and deployment`の`Source`を`GitHub Actions`にします。
4. GitHub Desktopで今回の変更をCommitし、`Push origin`を押します。
5. `Actions`タブで「Deploy game to GitHub Pages」が緑のチェックになるまで待ちます。
6. 公開ページを開き、タイトルから最終結果まで動作を確認します。

## 使用技術

- Vite：開発サーバーと公開用ビルド
- TypeScript：型によって間違いを見つけやすくするプログラミング言語
- Three.js：3Dマップ、背景、主人公、サイコロの描画
- CanvasTexture：外部画像に頼らず、章の地面模様やマス面をコードから生成
- GitHub Actions：Push後のビルドとGitHub Pages公開を自動化

## 主なフォルダ

- `AI/`：確定仕様、イベント・カード・バランス、開発計画、変更履歴、シミュレーション報告
- `game/src/data/v2/`：100マス版のマス、イベント、カード抽選、選択肢データ
- `game/src/game/v2/`：100マス版のゲーム進行処理
- `game/src/three/`：3Dマップ、章背景、装飾、主人公、サイコロ
- `game/src/simulation/`：100マス版を自動再現するバランス確認処理
- `.github/workflows/`：GitHub Pagesの自動公開設定

## 注意事項

- セーブ機能はありません。ページを再読み込みすると最初からになります。
- スマートフォンは横画面に対応していますが、PCブラウザを優先しています。
- 決済やリアル商品の購入処理はありません。
- `AI/`フォルダも公開リポジトリから閲覧できます。秘密情報や個人情報は記載しないでください。
