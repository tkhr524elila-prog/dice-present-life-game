# サイコロを振ってプレゼントが決まる、人生ゲーム的なゲーム

サイコロを振って3Dの人生マップを進み、現実的な人生イベントや選択を体験する1人用ブラウザゲームです。ゴールでは、歩んだ人生に応じた現金額とリアル景品が決まります。

## 公開ページ

[ゲームをブラウザで遊ぶ](https://tkhr524elila-prog.github.io/dice-present-life-game/)

初回公開前、またはGitHub Pagesの処理中はページが表示されないことがあります。公開方法は後述の「GitHub Pagesへの公開」を確認してください。

## 遊び方

1. 「ゲームをはじめる」を押します。
2. 「サイコロを振る」を押して、出目の数だけマップを進みます。
3. イベントや人生の選択を確認し、「次へ」で進めます。
4. プレゼント抽選で獲得したライフカードや、人生ノートを確認できます。
5. マス60へ到達すると人生が精算され、最終現金額と景品一覧が表示されます。

## 必要なもの

- PCブラウザ（優先環境）
- または横画面にしたスマートフォン
- ローカル開発を行う場合はNode.js 22とnpm

## ローカルで起動する方法

ターミナルでリポジトリの`game`フォルダへ移動し、次のコマンドを実行します。

```bash
cd game
npm install
npm run dev
```

表示されたURL（通常は `http://localhost:5173/dice-present-life-game/`）をブラウザで開きます。終了するときは、起動中のターミナルで `Control + C` を押します。

## ビルドと公開前確認

ビルドは、開発用コードをブラウザ公開用のファイルへまとめる処理です。

```bash
cd game
npm run build
```

成功すると`game/dist`へ公開用ファイルが作成されます。作成したファイルを公開に近い状態で確認する場合は、続けて次を実行します。

```bash
npm run preview
```

## 自動シミュレーション

同じ乱数シード（再現用の番号）で多数のプレイを再現し、現金・景品・ステータスのバランスを確認できます。

```bash
cd game
npm run simulate
```

現在の標準設定では、固定32パターンを各1,000回、ランダム選択を10,000回、合計42,000回実行します。

## GitHub Pagesへの公開

`.github/workflows/deploy.yml`が、`main`ブランチへのPush後に自動でビルドと公開を行います。

初回だけ、GitHubのリポジトリ画面で次を確認してください。

1. `Settings`を開きます。
2. 左側の`Pages`を開きます。
3. `Build and deployment`の`Source`を`GitHub Actions`にします。
4. GitHub Desktopで今回の変更をCommitし、`Push origin`を押します。
5. GitHubの`Actions`タブで「Deploy game to GitHub Pages」が緑のチェックになるまで待ちます。
6. 公開ページを開いて動作を確認します。

## 主な技術

- Vite：開発サーバーと公開用ビルドを担当
- TypeScript：型を使って間違いを見つけやすくするプログラミング言語
- Three.js：ブラウザに3Dマップ、主人公、サイコロを描画
- GitHub Actions：Push後のビルドとGitHub Pages公開を自動化

## 主なフォルダ

- `AI/`：確定仕様、データ設計、開発計画、変更履歴、シミュレーション報告
- `game/src/`：ゲーム本体のTypeScriptとCSS
- `game/src/data/`：マス、イベント、カード、選択肢などのデータ
- `game/src/simulation/`：ゲーム進行を自動再現するバランス確認用処理
- `.github/workflows/`：GitHub Pagesの自動公開設定

## 注意事項

- セーブ機能はありません。ページを再読み込みすると最初からになります。
- 1回のプレイ時間は45～60分程度を想定しています。
- スマートフォンは横画面に対応していますが、PCブラウザを優先しています。
- `AI/`フォルダも公開リポジトリから閲覧できます。秘密情報や個人情報は記載しないでください。
- 決済やリアル商品の購入処理はありません。
